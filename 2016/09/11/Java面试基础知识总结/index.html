Java面试基础知识

Posted by Cai Lei on 2016-09-11

此片文章主要是参考一片面经这里,进行的部分解答，最初的手写稿在2016年的3月和4月，最近重新整理准备相关的校招面试，发布在此。

Java基础：

  1. 九种基本数据类型的大小以及它们的封装类。
基本类型 位数 字节数 包装类
int 32 4 Integer
short 16 2 Short
long 64 8 Long
float 32 4 Float
double 64 8 Double
boolean true/false 1或者4 Boolean
char 16 2 Character
void Void

总结：16位的有char,short. 32位的float，int 。64位的double,long.关于boolean请见这篇，

  1. equals()与==区别

    （1）基本类型，==是比较值是否相等

    （2）复合类型，==比较地址（引用）。Object类中的equals方法内部是==实现的也就是说默认的equals是比较二个对象引用是否相等。而许多类，如String，Integer等包装类，Date类覆盖了Object类的这个equals方法。String的equals就是比较二个字符串值是否相等了。参考链接

  2. Object有哪些公用方法？

    clone,equals,hashcode,notify,notifyAll,finalize,wait(有无参数),toString。

  3. Java的四种引用强软弱虚

    强：使用最普遍的引用，如果一个对象是强引用的，那么GC绝不会回收，当内存空间不够，宁愿报OOM Error异常，终止程序，也不会回收强引用来解决内存空间不够的问题。

    软：Soft，内存空间够则不回收，不够则回收。用来实现内存敏感的高速缓存。可以和引用队列联合使用。

    弱：Weak，更短暂的生命周期，GC管辖内存区域过程中，一旦发现弱引用对象则回收之。不管内存是否够。

    虚：Phantom，顾名思义，形同虚设，并不决定对象的生命周期，仅持虚引用和没有一样。任何时候都可能被回收，用途：跟踪对象被垃圾回收器回收的活动，虚引用必须和引用队列ReferenceQueue联合使用，GC回收一个对象发现还有虚引用，那么回收内存前会把其关联到引用队列 。

  4. Hashcode的作用：

    函数返回散列值，也就是根据一定的规则把与对象有关的信息映射成为一个散列值，在集合中添加元素可以先看该元素的散列值，然后减少equals方法的调用次数，提高程序的效率，提高寻找对象的作用。

    两个对象相同则hashcode一样，两个hashcode相等，不一定说明它们是一个对象。但是hashcode不同必定二个对象是不同的。

    注意：重写equals方法必须也要重写hashcode方法！

  5. ArrayList、LinkedList、Vector的区别

    三者都实现了java.util.List接口

    一、同步性：ArrayList、LinkedList不同步，非线程安全，Vector线程安全，同步、性能也就相对较低。

    二、数据增长：ArrayList增长为50%，Vector增长为原来的一倍因此适合大量的数据。

    三、检索与删除、增加性能：ArrayList/Vector在末尾增删复杂度O(1),检索指定的位置复杂度O(1),在其他地方增删复杂度O(n-i),之后的都要移动

    ​ LinkedList:检索性能低（因为链标表的原因），O(i)增删的效率高（仅仅改变指针的指向）O(1)

    四、实现方式：ArrayList/Vector类似数组的方式。LinkedList是双向链表方式。

  6. String、StringBuilder、StringBuffer的区别

    String不可变字符串，每次都是新建一个字符串，然后指针指向这个字符串。StringBuilder是线程不安全可变字符串的、StringBuffer是线程安全的可变字符串类。因为String每次都是新建字符串所以效率大多数低于StringBuffer.

    String s = “This”+”is”+”a String”;为什么比StringBuffer快呢？这是JVM对这种字符串字面值的拼接的优化。相当于一个“This is a String”。String s = s1+s2+s3的方式则是按原来的方式（新建字符串）的方式。比StringBuffer的append拼接慢。

  7. Map、Set、List、Queue、Stack特点

    Map是键值对存储。

    Set是不包含重复元素的集合。

    List(列表)： List的特征是其元素以线性方式存储，集合中可以存放重复对象。

    Queue是先进先出的队列。

    Stack后进先出的栈。

  8. HashMap、HashTable区别

    HashMap是线程不安全、非同步的，键值允许为空（null）的，HashTable是线程安全、同步的、键值不允许为空的。

  9. HashMap、ConcurrentHashMap的区别

    HashMap是线程不安全的，ConcurrentHashMap是线程安全的版本。HashTable的锁实现是对对象整体加锁。ConcurrentHashMap基于lock操作，锁住的不是整个对象。

  10. TreeMap、HashMap、LinkedHashMap区别

    LinkedHashMap保存了插入的顺序。输出得到的输出顺序与输入一样，HashMap是随机存储。TreeMap实现了SortMap接口，按键值的升序排列。
